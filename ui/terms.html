Пользовательское соглашение

ИРУ — Интеллектуальный Режим Управления · экспериментальный beta-stage сервис

1. Общие положения

Настоящее соглашение регулирует использование ИРУ. Используя сервис, вы подтверждаете согласие с актуальной редакцией документа.

ИРУ предоставляется по модели as is. Сервис находится в активной разработке и не заявляется как production-grade enterprise security system.

2. Что делает ИРУ

ИРУ принимает задачу пользователя, выбирает инструменты и отправляет действия локальному агенту, установленному на устройстве пользователя. Агент выполняет действия на устройстве и возвращает результат серверу.

Ответы о состоянии устройства должны опираться на текущие tool results, но система не гарантирует абсолютную безошибочность интерпретации, планирования и выполнения.

3. Ответственность пользователя

  • не использовать ИРУ для несанкционированного доступа к чужим устройствам и данным;
  • не использовать сервис для вредоносного ПО, обхода защиты или незаконных действий;
  • не передавать токен доступа третьим лицам;
  • контролировать последствия задач, которые пользователь запускает через ИРУ;
  • проверять результаты перед использованием созданных файлов, скриптов и документов.

4. Данные и локальное состояние

Сервер может хранить учётные данные, историю чатов, задачи, технические summaries, audit log и настройки пользователя. Локальный агент хранит состояние устройства в своём `IRU_HOME`, включая activation/runtime/state/passport cache.

Отдельные данные могут использоваться для улучшения качества системы только в рамках пользовательского согласия и настроек сервиса.

5. Риски выполнения

ИРУ выполняет реальные действия на устройстве. Пользователь отвечает за задачи, которые инициирует, и должен понимать последствия операций с файлами, приложениями, shell-командами и документами.

Разработчик не несёт ответственности за ущерб, вызванный некорректной формулировкой запроса, ошибкой модели, локальным сбоем, проблемой оборудования, повреждением пользовательских данных или компрометацией токена по вине пользователя.

6. Ограничения

Разработчик вправе ограничивать доступ к сервису, менять лимиты, отключать рискованные возможности, обновлять агента и сервер, а также временно приостанавливать работу функций для безопасности или обслуживания.

7. Изменение условий

Актуальная версия соглашения публикуется по адресу /terms. Продолжение использования сервиса после обновления текста считается согласием с новой редакцией.
